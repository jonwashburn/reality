TITLE; Entropy is an Interface: Reversibility Below, Irreversibility at Commit
ONE_LINER; Entropy is recognition-relative code length; the second law lives at the readout channel, not in the substrate.

C2; Maxwell's demon pays at commit: ΔWork ≥ k_B T ln 2 · ΔS_bits; reversible bulk steps do not alter S absent recognition commits.
C3; Changing the window/kernel (W,K) changes measured S in a law-governed way; aligned 8-tick windows preserve invariants (no spurious entropy).

FOUNDATIONS (math, minimal)
F1; Entropy as MDL @ channel: S_{W,K}(μ) = L^*(p_Y), p_Y(y)=∫K(y|W(x)) dμ(x). Special cases: (i) Shannon/Hartley when code optimal over Y; (ii) Gibbs when p_Y is Gibbsian.
F2; Monotonicity: For any post-processing Y→Z via a stochastic map Q (data-processing), L^*(p_Z) ≥ L^*(p_Y) up to O(1); hence entropy increases under further coarse-graining.
F3; Arrow-of-time = commit order: A recognition pipeline Π = (W_1,K_1)→commit→(W_2,K_2)→commit… yields S_{Π} nondecreasing at each commit; bulk reversible steps with no commit keep S fixed.
F4; Landauer bridge: Reversible micro-evolution allows isothermal erasure cost bounded by k_B T ln 2 per bit of log-loss; identify ΔS_bits with MDL code surplus removed at commit.
F5; Eight-tick alignment: Using PatternMeasurement lemmas, if windows respect 8-tick alignment, block sums equal Z(w) and observed averages equal the invariant; entropy does not grow from mere realignment.

@PAPER_EVOLUTION
TITLE; Darwin as Minimum Description Length on a Ledgered Substrate
ONE_LINER; Natural selection is MDL: organisms that compress their ecologies most effectively—subject to recognition/energy budgets—win.

CLAIMS
E1; Fitness = code-length savings: For an environment E and genotype g (program P_g), fitness is proportional to negative codelength to maintain viability: F(g;E) ∝ −L(P_g,E).
E2; Variation is anisotropic: Mutational moves that reduce ledger overhead (respect continuity/cost J) are exponentially favored; "random wrt fitness" becomes "random inside iso-cost shells."
E3; Modularity emerges from shared subroutines: When tasks share structure, duplication+reuse lowers L; selection pressures modular, hierarchical architectures.

FOUNDATIONS (math, minimal)
F1; Environment as source: Let ℰ be a distribution over tasks/data; an organism implements a compressor–controller C_g with parameters θ_g. Define performance L_g = L(model) + L(errors | ℰ). Selection minimizes L_g under resource constraints.
F2; Replicator–MDL link: Suppose replication rate r(g) ∝ exp(−β·L_g). Then the stationary distribution π(g) ∝ exp(−β·L_g). Lower code wins; β encodes resource/temperature-like scarcity.
F3; J-geometry bias: Allowed phenotypic changes are constrained by cost J: steps that increase J sharply are pruned; small 1/x↔x symmetry around x=1 biases toward balanced, low-overhead designs (homeostasis).
F4; Information budget: I(g;ℰ) ≤ budget set by recognition length/time; improvements in mutual information per unit ledger cost drive adaptive steps.

CLASSICAL MAP
MAP; Darwin/Wallace selection ↔ MDL: "Most fit" = "shortest total description for viable behavior," echoing Akaike/BIC logic in model selection.
MAP; Fisher's theorem: Growth of mean fitness = decrease of average L_g given additive heritable components; reframed as code-length gradient flow.
MAP; Rate–distortion: Trade-off between metabolic/recognition resources (rate) and tolerable loss (distortion); organisms sit near Pareto fronts.

OPERATIONAL (archival-only demonstrations)

D3; Behavioral ecology datasets (foraging, navigation): demonstrate cross-task transfer lowering L for species with richer internal models.

DR2; Modularity theorem (sketch): If ℰ decomposes into shared sub-tasks with mutual information M, adding a reusable module of size b bits saves ≥M−b; selection favors modules when M>b.
